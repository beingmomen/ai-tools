# Dynamic Context Pruning (DCP)

## 📌 Overview
**Dynamic Context Pruning (DCP)** هو Plugin لـ OpenCode بيهدف إلى تقليل استهلاك التوكنز (tokens) وتحسين أداء النماذج الذكية (LLMs) عن طريق إدارة الـ context (المحادثة) بشكل ذكي.

بدل ما يتم إرسال كل تاريخ المحادثة للنموذج، DCP بيقوم بضغط وتنظيف المحتوى مع الحفاظ على المعلومات المهمة.

---

## 🚀 المشكلة
عند استخدام LLMs في جلسات طويلة:
- حجم الـ context بيكبر جدًا
- التكلفة بتزيد
- الأداء بيقل
- النموذج ممكن يهلوس أو يفقد التركيز

---

## 💡 الحل
DCP بيقوم بـ:
- ضغط المحادثات القديمة (Compression)
- حذف التكرار (Deduplication)
- تنظيف الأخطاء (Error Pruning)

---

## ⚙️ How It Works

### 🗜️ 1. Compression
- بيحول أجزاء من المحادثة إلى **ملخصات ذكية**
- بيستبدل المحتوى الطويل بملخص صغير
- لا يتم حذف البيانات الأصلية، فقط يتم تقليلها أثناء الإرسال

#### Modes:
- **Range Mode (default):** ضغط مجموعة رسائل معًا
- **Message Mode (experimental):** ضغط كل رسالة بشكل مستقل

---

### 🔁 2. Deduplication
- إزالة التكرار في استدعاءات الأدوات (tools)
- الاحتفاظ بأحدث نتيجة فقط

---

### ❌ 3. Purge Errors
- حذف مدخلات الأدوات الفاشلة بعد عدد معين من الرسائل
- الاحتفاظ برسائل الخطأ فقط

---

## 🤖 التشغيل (Automatic vs Manual)

### ✅ الوضع الافتراضي (Automatic)
- يعمل تلقائيًا بدون تدخل
- النموذج يقرر متى يتم الضغط والتنظيف

### 🕹️ الوضع اليدوي (Manual)
يمكن تفعيله من الإعدادات:
```json
{
  "manualMode": {
    "enabled": true
  }
}
